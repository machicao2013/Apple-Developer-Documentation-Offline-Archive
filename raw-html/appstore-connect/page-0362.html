Challenges

Learn how to configure challenges in App Store Connect.

Challenges properties

Properties

Description

Reference Name

An internal name that you must provide for each challenge. It can be up to 255 alphanumeric characters and spaces. This is the name that you use if you search for your challenges in App Store Connect.

Challenge ID

A chosen alphanumeric identifier for your challenge. It can be up to 100 alphanumeric characters, spaces, and periods (assumes single-byte characters). Your challenge ID is a permanent setting and can't be changed.

Repeatable

For repeatable gameplay, players can choose from the options 1 try, 3 tries, and unlimited tries. For instance, if you select "3 tries" for Repeatable and "1 day" for Duration, the player with the highest score in 3 rounds within a 24-hour countdown wins.

For non-repeatable gameplay, players can only attempt a challenge with 1 try.

Associated Leaderboard

The selected leaderboard will be used as the basis for challenge.

Default image

An image that represents your challenge across all localizations. It appears in Game Center when someone is playing the challenge. The image must be a .jpeg, .jpg, or .png file that’s 3840 x 2160 pixels, at least 72 ppi, no transparency and in the RGB color space.

Challenge language properties

Challenge language properties collect language-specific text used in challenge’s display. You must include these properties for at least one language.

Property

Description

Localization

The language in which your challenge appears. There must be at least one language defined.

Display Name

The display name of the challenge in the specified language. For example, if you choose Finnish from the Language menu, enter the Finnish name for the challenge.

Description

A short description that helps players understand the details of the challenge.

Image

A localized image that represents the challenge. The image must be a .jpeg, .jpg, or .png file that’s 3840 x 2160 pixels, at least 72 ppi, no transparency and in the RGB color space. If you don't provide this image, the default image will display.

Challenge statuses

Challenge statuses indicate whether the challenge has been approved and made live with the app.

Status

Description

Prepare for Submission

You’ve created a Game Center component in App Store Connect but you're still preparing information for it.

Ready for Review

You’ve entered all of the required metadata for your Game Center component and indicated that you intend to submit it for review, but haven’t submitted it yet.

Learn how to submit Game Center components for review.

Waiting for Review

Apple received your submission, but hasn’t started the review.

While you're waiting for the review, you can remove your Game Center component from submission.

In Review

App Review is reviewing your Game Center component.

Accepted

This Game Center component has been accepted, but one or more items in your submission were rejected.

The component can only publish when all items in the submission are accepted. You can edit the rejected items and resubmit, or remove them from the submission to move forward with the currently accepted items.

Live

Your Game Center component has been accepted and is ready for distribution.

Rejected

Your Game Center component wasn't accepted. App Review notifies App Store Connect users with the Admin, App manager, or Developer role with the issues.

Learn how to reply to App Review messages.

Developer Rejected

You removed your Game Center component from review. When you’re ready, resubmit your component.
