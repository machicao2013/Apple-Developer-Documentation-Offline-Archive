Provide export compliance information for beta builds

Specify encryption use for your build to avoid the beta being marked as Missing Compliance. Answer the export compliance questions or attach previously approved documentation in the TestFlight section. View role permissions.

  1. In Apps, select the app you want to view.

  2. Click the TestFlight tab.

  3. In the sidebar, under Builds, click the platform.

  4. In the Build column in the table on the right, click the app icon or build string for the build that’s missing compliance information.

    To address compliance issues for a build, you can either click Manage beside the relevant build to answer the required questions, or click Choose File to upload previously approved documentation.

  5. Click Provide Export Compliance Information, and answer the required questions.

  6. If app encryption documentation is required, click Go to App Encryption Page to upload your file. Apple will provide you with a key value to enter in Xcode This will remove the need to answer encryption questions with each app submission. Learn more about uploading app encryption documentation.

    If documentation isn’t required, click Save. You can indicate that your app doesn’t use encryption or is exempt from documentation by specifying encryption settings in the app's Info.plist file in Xcode. This eliminates the need to answer encryption questions with each app submission.
